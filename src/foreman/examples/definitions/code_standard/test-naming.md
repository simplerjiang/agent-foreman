# 代码规范：测试命名（test-naming）

> 测试该怎么放、怎么取名。**通用脱敏示例**，请按你的测试框架与习惯改写。

## 文件与位置
- 测试放在 `tests/`，文件名 `test_<被测模块>.py`（与 pytest 默认发现规则一致）。
- 一个被测模块对应一个测试文件，别把不相关的测试堆一起。

## 函数命名
- 测试函数以 `test_` 开头，名字说清「场景 + 期望」：
  - `test_login_rejects_wrong_password`
  - `test_parse_empty_body_fails_closed`
- 一个测试只验一件事；要验多个分支就拆多个函数，别用一个巨型测试。

## 写法
- 结构按 **准备 → 执行 → 断言**（arrange / act / assert）三段清楚分开。
- 断言带含义，失败信息能自解释；避免只 `assert result` 这种看不出期望的断言。
- 不依赖测试间的执行顺序，不共享可变全局状态。
