## 问题清单

### 问题 #01 — 无单实例锁，重复启动会拉起第二个独立实例 🟠 Bug

- **现象**：在 App 已运行的情况下再次启动 `foreman.exe`（等价于用户在桌面/任务栏二次双击图标），会**直接拉起第二个完全独立的进程 + 窗口**，而不是聚焦已有窗口。第二个窗口启动后白屏数秒，随后渲染出与第一个实例**完全相同的会话数据**（因为连的是同一个 `foreman.db`）。
- **进程证据**：启动两次后 `foreman.exe` 进程数从 2 个变成 4 个（每个实例 = 1 窗口进程 + 1 引擎子进程）。
- **为什么是问题**：Foreman 是一个会**驱动本地编码 agent、对工作区做 git checkpoint、并发下发命令**的监督型工具。两个实例同时连同一个 `foreman.db`、可能同时驱动同一批 agent / 抢同一个工作区，存在数据竞争与「双重驾驶」风险（重复 checkpoint、状态错乱）。对桌面 App 来说，二次双击图标弹出重复窗口本身也很迷惑。
- **建议**：加单实例锁（命名互斥量 / 端口占用探测 / lock 文件）。检测到已有实例时，**聚焦已存在的窗口**而不是新开一个。这是桌面 App 的基本预期行为。
- **截图**：`screenshots/01-second-instance.png`

### 问题 #02 — 第二实例启动时整窗白屏数秒、无加载提示 🟡 UX

- **现象**：第二个实例（以及冷启动）会有数秒**纯白窗口**，窗口内没有任何 splash / 骨架屏 / loading 指示，看起来像卡死。
- **为什么是问题**：本地引擎 + WebView2 冷启动需要时间，但用户看到的是「白屏」而非「正在加载」。对首启体验不友好，容易让人以为程序挂了。
- **建议**：窗口内嵌一个轻量 loading 态（logo + 一行「正在启动本地引擎…」），引擎 ready 后再切到主界面；或让原生 splash 覆盖到主界面可交互为止。
- **截图**：`screenshots/01-second-instance.png`（注：成功捕获到白屏到渲染的中间态见对话记录）
