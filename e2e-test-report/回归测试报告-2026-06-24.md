### N1 — 侧边栏会话列表状态用英文，与头部中文徽标不一致 🟡 Minor 🖱️
- **现象**：左侧会话列表项显示 `pm-agent · cancelled · 16:..` / `· done ·` / `· running ·`（英文原始 status），而会话头部徽标是中文「已取消/完成/运行中」。同一状态两套语言。
- **影响**：中文用户在列表里看到英文状态词，和头部不一致，违背 i18n 一致性。
- **建议**：列表项状态也走 i18n 字典（done→完成 / cancelled→已取消 / running→运行中 / failed→失败），与头部统一。

### N2 — 统一错误组件覆盖不全，多条后端错误仍泄漏原始码/英文 🟠 Major 🔎
- **现象**：`friendlyError` 映射缺 `no_context`（压缩上下文）、`no_store`/`session_not_found`/`decline`（简报/下发）、以及 fetch 网络错误；这些会原样显示（如"压缩失败: no_context"、英文 "Failed to fetch"）。
- **影响**：#6/#17 只做到"组件统一"，没做到"消息齐全"，部分主链路失败仍不是人话。
- **建议**：把后端各路由实际可能返回的 error code 全量补进 `friendlyError` 映射，并给 fetch/网络错误一个兜底人话（"网络异常，请重试"）；可加单测覆盖"每个 error code 都有中文映射"。

### N3 — #4 摘要 skip 列表只覆盖英文开场白 🟡 Minor 🔎
- **现象**：`firstSubstantiveLine` 的跳过正则是固定英文清单；"Let me…""Sure,…""好的，我来…"等开场白不会被跳过。
- **建议**：清单补常见中英文开场白，或改为"跳过明显的元话语/思考行"的更稳健启发式。

### N4 — #2 流式 PM 文本未在代码层强制中文 🟡 Minor 🔎
- **现象**：PM 流式思考/状态文本展示模型原文，仅靠软指令约束语言；模型偶尔不遵守时仍可能出现英文。
- **建议**：要么在渲染层对 PM 状态流做语言兜底/提示，要么把"输出语言"作为更硬的约束（系统级），避免再现上一轮"一屏英文黑话"。

### N5 — WebView2 内容被标准屏幕捕获抓成全黑（远程/录屏隐患）🟠 Major（方法学+产品）
- **现象**：Foreman 窗口内容**用户肉眼正常**，但 **computer-use 截图 / `PrintWindow`（前台时）抓到的是纯黑**；只有 GDI `CopyFromScreen`（前台且已绘制时）能抓到。经隔离实验：用一个**最小 pywebview 窗口**（无 splash、无 DPI 调用、非打包 exe）即可复现"工具全黑、CopyFromScreen 正常"——**这是 WebView2 GPU/DirectComposition 渲染的通用现象，与 codex 本次改动无关**（排除了 DPI 调用、`SetWindowDisplayAffinity` 等假设）。
- **影响**：对"桌面级 + **远程** agent 控制器"，若远程投屏/录屏/远程桌面用的是 GPU 表面抓不到的捕获路径，Foreman 窗口可能**显示为黑屏**。属潜在风险，建议验证主流投屏/录屏工具下是否正常；必要时为 WebView2 配置可被捕获的渲染路径（如禁用 GPU 合成的兜底选项）。
- 备注：本轮我因此改用 `CopyFromScreen`+裁剪+坐标标定来"看屏"，故 GUI 点击较慢——见末尾方法学。

### N6 — #9 只有失败终态、无重试 🟠 Major（健壮性）🔎
- 失败时会话进入 `failed` 终态（好），但没有"重试"按钮；端点抖动后用户只能删了重发。建议补一键重试（复用同一 goal）。

### N7 — 工作区仍是整盘 E:\ + 工具全开（用户自配，护栏已加）🔵 建议
- 护栏黄条已加（#13 ✅），但当前配置仍是整盘 + full access + autonomy 可上调的高风险组合。属用户自决，提醒留存。

## 附：方法学（为什么这轮 GUI 点击偏慢 + 并行 workflow 怎么帮上忙）
- **截图全黑（N5）**逼我放弃 computer-use 自带截图，改用 PowerShell `CopyFromScreen` 全屏抓 → 裁剪放大读小字 → 用实测标定的比例（actual_px = tool×1.3187）反算 computer-use 点击坐标。每一步都要往返，且坐标偶有估偏需重试——这是慢的主因，**并行多 agent 也救不了**（一个窗口、一个光标、一个前台焦点，改状态的点击天然串行；多 tab 打同一后端还会互相踩共享 SQLite）。
- **真正的提速来自并行 workflow**：把"代码级+API 级回归核验"拆成 6 组**真并行**跑（约 4 分钟、6 agent），独立交叉验证手测结论，并补出纯靠点击看不到的代码缺口（如 #6/#17 的映射不全、#4 的英文-only skip、#2 的流式未强制）。
- **沉淀**：已据此把 `foreman-e2e` skill 升级为"**并行代码审计 + 串行真机点击**"的混合流程（见 skill 更新）。
